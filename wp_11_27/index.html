图论

  • 多人去同一个地方（多起点 -> 单终点）
    • 题目特征：题目给了 NNN 个人（或者 NNN 个城市），问：“这 NNN 个人都要去同一个目的地 TTT 集合，请问谁离目的地最远/最近？或者所有人走的总路程是多少？”
    • 为什么不能直接做？如果你按照正常的思维：每个人跑一次 Dijkstra，就要跑 NNN 次。时间复杂度爆炸，绝对超时 (TLE)。反向建边怎么破？
    • 逻辑： “AAA 走到 BBB” 等价于 “在反向图中，从 BBB 走到 AAA”。
    • 操作：建反图（所有箭头掉头）。只从目的地 TTT 出发，在反向图上跑一次 Dijkstra。跑出来的 dist[i]，意思就是 “原图中 iii 号点走到目的地 TTT 的最短距离”。
    • 收益： 跑 NNN 次变成跑 1 次，秒杀。
  • 有去有回（往返最短路）
    • 题目特征：题目问：“邮递员从起点 111 送信到 NNN 个地方，送完还得回到起点 111，求最小的总路程。”或者简单的问：“求 AAA 到 BBB 再回到 AAA 的最短路径。”
    • 反向建边怎么破？去程： 在正向图上跑一次 Dijkstra（起点 111），得到 dist_to[i]（从家去别处的距离）。
    • 回程： 在反向图上再跑一次 Dijkstra（还是从起点 111 开始跑！），得到 dist_back[i]。
    • 注意：反向图上从 1 出发到 i 的距离，其实就是原图里 i 回到 1 的距离。
    • 答案： 第 iii 个地点的往返总路程 = dist_to[i] + dist_back[i]。

小鸡鸡桥

  • 断环为链 (Breaking the Circular Array)
    • 什么时候用？题目特征： 题目说“大家围成一个圈坐着”、“项链上的珠子”、“环形跑道”。
    • 难点： 数组是直的，环是圆的，处理“下标越界”或者“跨过终点回到起点”非常麻烦，容易写全是 Bug。
    • 神操作：复制一遍！ 把原数组 a[1…n] 复制一份接在后面，变成 a[1…2n]。比如原数组是 [1, 2, 3]，新数组变成 [1, 2, 3, 1, 2, 3]。
    • 效果： 环上任意一段长度为 NNN 的路径，现在都在这个 2N2N2N 的直数组里变成了连续的一段。
    • 做法： 以后再也不用 % n 取模了，直接在 1 到 2n 的范围里做滑动窗口或者前缀和。

  • 套路二：区间修改 + 求单点值 (RUPQ)这是最巧妙的类型，用到了差分思想。
